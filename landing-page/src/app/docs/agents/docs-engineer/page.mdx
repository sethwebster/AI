# Docs Engineer (Sam)

Implementation-grade documentation focused on accuracy, clarity, completeness, and developer usability.

## Overview

**Agent Type:** `docs-engineer`
**Persona:** Sam
**Expertise:** Clear, functional documentation

Sam is your go-to for everyday documentation. Documentation that's accurate, complete, and gets developers up and running quickly.

---

## When to Deploy

Use Docs Engineer:

- After adding new features
- When creating API documentation
- For user guides and tutorials
- When updating README files
- For architecture documentation

**Proactive usage:** Document as you build, not after you ship.

Sam excels at:

- **Accuracy** - Every detail verified and correct

- **Completeness** - All parameters, options, edge cases covered
- **Developer usability** - Quick starts, code examples, common workflows
- **Searchability** - Clear headings, logical structure

## What Makes Good Documentation

### Structure

- **Prerequisites** - What users need before starting
- **Quick Start** - Get to success in less than 5 minutes
- **Concepts** - Build understanding progressively
- **Reference** - Complete API surface
- **Examples** - Runnable code samples

### Code Samples

- **Runnable** - Not toy examples, real code
- **Complete** - Include imports, setup, expected output
- **Commented** - Explain non-obvious logic
- **Tested** - Actually works

### Style

- **Active voice** - "Click the button" not "The button is clicked"
- **Present tense** - "Returns user" not "Will return user"
- **Scannable** - Short paragraphs, clear headings
- **Accessible** - Works with screen readers

4. Ship
